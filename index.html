Application de Gestion de Budget Personnel - BudgetManager

1. Présentation Générale du Projet

Nom du projet : BudgetManager

Type d'application : Application console développée en Java

Paradigme : Programmation Orientée Objet (POO)

BudgetManager est une application permettant à un utilisateur de gérer efficacement ses revenus et ses dépenses. Elle vise à offrir un outil simple, structuré et automatisé pour assurer un meilleur contrôle financier personnel.

2. Contexte et Justification

La gestion financière personnelle constitue un enjeu majeur pour les étudiants et les jeunes professionnels. L'absence de suivi des dépenses entraîne souvent des déséquilibres budgétaires.

Le projet BudgetManager répond à ce problème en proposant un système permettant :

  • L'enregistrement structuré des revenus
  • L'enregistrement catégorisé des dépenses
  • Le calcul automatique du solde
  • L'affichage synthétique de la situation financière

3. Objectifs du Projet

3.1 Objectif Général

Concevoir une application orientée objet permettant la gestion complète d'un budget personnel.

3.2 Objectifs Spécifiques

  • Mettre en œuvre les principes fondamentaux de la POO
  • Implémenter l'héritage et le polymorphisme
  • Structurer les données à l'aide de collections (ArrayList)
  • Automatiser les calculs financiers

4. Périmètre du Projet

Inclus dans le projet :

  • Ajout de revenus
  • Ajout de dépenses
  • Catégorisation des dépenses
  • Calcul du solde
  • Affichage des transactions

Non inclus :

  • Connexion à une base de données externe
  • Interface graphique avancée
  • Système de sécurité avancé

5. Exigences Fonctionnelles

Code Description
EF1 Permettre l'ajout d'un revenu
EF2 Permettre l'ajout d'une dépense
EF3 Afficher la liste des transactions
EF4 Calculer automatiquement le solde
EF5 Afficher un résumé financier

6. Exigences Non Fonctionnelles

  • Performance : exécution rapide sans latence perceptible
  • Maintenabilité : code structuré et modulaire
  • Lisibilité : respect des conventions Java
  • Extensibilité : possibilité d'ajouter une base de données ultérieurement

7. Architecture et Modélisation Objet

Classes principales :

  • Utilisateur
  • Transaction (classe abstraite)
  • Revenu (hérite de Transaction)
  • Depense (hérite de Transaction)
  • BudgetManager

Principes POO utilisés :

  • Encapsulation
  • Héritage
  • Abstraction
  • Polymorphisme

8. Contraintes Techniques

  • Langage : Java
  • IDE recommandé : Eclipse
  • Stockage : Mémoire vive (ArrayList)
  • Version Java : 17 +

10. Conclusion

Le projet BudgetManager constitue une application pédagogique complète permettant de mettre en pratique les concepts fondamentaux de la Programmation Orientée Objet en Java. Il répond à un besoin réel de gestion financière tout en respectant une architecture logicielle claire, modulaire et évolutive.
